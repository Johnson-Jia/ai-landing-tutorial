# 协调交接模板包

团队在 AI 转型中的交接与协作模板——把"部门间怎么对接"从口头约定变成可照做的标准。
对应教程：[阶段 4 闭环试点](../../pages/stage4.html)、[阶段 5 推广度量](../../pages/stage5.html)

## 为什么需要
AI 让代码产出提速，但**交接不畅会把提速吃掉**：开发用 AI 生成的代码，测试不知道怎么验收；前后端各用 AI 生成，接口对不上；跨部门变更没有统一交接，上线互相踩坑。这套模板把高频交接场景标准化。

## 模板清单
| 模板 | 场景 | 核心约定 |
|---|---|---|
| [开发-测试交接.md](./开发-测试交接.md) | 开发 → 测试 | AI 生成代码的交付物 + 验收标准 + 防偷改测试 |
| [前后端协作.md](./前后端协作.md) | 前端 ↔ 后端 | 接口契约先行 + Mock 并行 + 契约变更通知 |
| [跨部门交接.md](./跨部门交接.md) | 研发 ↔ 运维/安全/业务 | AI 变更的上线交接 + 知识资产共享 |
| 产品 ↔ 研发 | 产品 → 研发 | 已在 [产品手册 §3.3](../role-handbooks/产品手册.md)（spec 驱动需求交接），此处不重复 |

## 使用方式
1. 按场景挑对应模板
2. 把模板里的"交接物 / 验收标准 / 检查项"落进自己的工作流（提交规范、Issue 模板、协作文档）
3. 关键条款用工具强制（如"防偷改测试"用 Hook，不只是写在文档里）

## 与其他模块的关系
- 配合 [角色操作手册](../role-handbooks/)（各角色知道自己的交接职责）
- 配合 [验收审查清单](../review-checklists/)（交接后的验收标准）
