# 角色操作手册包

5 个角色的 AI 转型操作手册（开发 / 测试 / 组长 / 产品 / 运营），覆盖转型全员（含非技术岗）。
对应教程：[《角色操作手册》](../../assets/ref/角色操作手册.html)

## 手册清单
| 手册 | 角色 | 核心任务 |
|---|---|---|
| [开发手册.md](./开发手册.md) | 开发 | 精通 Claude Code + CLAUDE.md + impact 影响分析 + 人审 AI 产出 + 沉淀 Skill |
| [测试手册.md](./测试手册.md) | 测试 | 录制转生成 + 测试资产规范 + AI 占比度量 + 防偷改测试 |
| [组长手册.md](./组长手册.md) | 组长 | 全员培训 + 规范资产化 + 带试点 + 数据汇报 |
| [产品手册.md](./产品手册.md) | 产品 | Codex 做原型 + 需求结构化 + 与开发 AI 协作 |
| [运营手册.md](./运营手册.md) | 运营 / 行政 / HR | 网页版 AI 处理文档/表格/汇报（不碰命令行）+ 5 个核心场景 prompt + 防幻觉核对 |

## 使用方式
各级人员**拿自己那份手册照做**：每份含 6 个部分——
1. 角色定位（AI 时代这个角色怎么变）
2. 核心任务（要做的几件事）
3. 怎么做（按优先级的步骤）
4. 工具箱（用什么 + 链接到教程/demo）
5. 检查清单（自检是否到位）
6. 常见坑（避免踩雷）
