# 组长 · AI 转型操作手册

> 对应教程：[阶段 1-6](../../pages/stage0.html)（贯穿）、[汇报材料模块](../../assets/ref/汇报材料-AI生成指南.html)
> 一句话：你是转型的基层推动者——把战略变成团队动作，把效果变成可汇报的数据。

---

## 一、你的角色定位
组长是 AI 转型落地的**关键节点**：上接战略，下带团队。战略再好，组长不推就落不了地。你的核心价值是「**推动 + 赋能 + 度量 + 汇报**」。

## 二、核心任务
1. 组织全员培训到 **100%**（不留死角）
2. 推动**规范资产化**（CLAUDE.md / Skill / Hook 提交 Git）
3. 带头**试点**（跑通闭环，拿数据做示范）
4. 收集**效果数据**，向上汇报

## 三、怎么做

### 3.1 全员培训 100%
- 直播动员 + Claude Code 安装使用 + AI 原理认知 + 实操练习 + 知识考核
- **留死角的团队推不动**——那 40% 不会用的人会拖后腿

### 3.2 规范资产化推动
- 定团队 CLAUDE.md 标准，要求人人提交 Git
- 高频操作做成 Skill 共享（团队 Skill 仓库）
- 关键规则用 Hook 强制（CLAUDE.md 只是建议，Hook 是确定性的）

### 3.3 带试点（阶段 4）
- 选 1-2 个真实需求，用 OpenSpec + Superpowers 跑通闭环
- 拿案例做示范，复制到全组
- **试点不拿数据，就推不动后续投入**

### 3.4 效果收集 + 汇报
- 用 [`demos/ai-metrics`](../ai-metrics/) 收集 AI 占比 / 产出 / Bug 数据
- 用 [汇报材料模板](../../assets/ref/汇报材料-AI生成指南.html)（周报/阶段总结）向上汇报
- **数据驱动，不靠感觉**

## 四、你的工具箱
| 工具 | 用途 |

| [汇报材料模板](../../assets/ref/汇报材料-AI生成指南.html) | 周报 / 阶段总结（向上汇报）|
| demos/ai-metrics | 效果度量（AI 占比/产出/Bug）|
| 团队 Skill 仓库 | 规范资产化 |

## 五、检查清单
- [ ] 全员培训覆盖率 100%
- [ ] 团队 CLAUDE.md 标准已定、人人提交
- [ ] 已带 1-2 个试点跑通闭环，有示范案例
- [ ] AI 占比 / 产出 / Bug 已度量，定期向上汇报

## 六、常见坑
- ❌ 培训留死角（部分人不会用，半年后差距拉大）
- ❌ 规范不沉淀（各搞各的，复制不开）
- ❌ 试点不拿数据（推不动投入，被认为是玄学）
