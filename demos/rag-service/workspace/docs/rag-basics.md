# RAG 检索增强生成原理

## RAG 是什么

RAG(Retrieval-Augmented Generation,检索增强生成)是一种结合外部知识检索与大模型生成的架构。它的核心思想是:在生成答案前,先从知识库中检索出与问题相关的片段,作为上下文喂给模型,从而让回答有据可依,显著降低幻觉。

## 标准流程

一个典型的 RAG 管线包含以下步骤:

1. **切分(Chunking)**:把长文档按语义/长度切成片段,通常 200-800 字符。
2. **Embedding**:用嵌入模型把每个片段编码成向量,存入向量库。
3. **检索**:用户提问时,把问题转成向量,在库中做近邻搜索,召回 Top-K 片段。
4. **重排(Rerank)**:用一个更精细的 cross-encoder 模型对召回结果重新打分,提升相关性。
5. **生成**:把重排后的 Top-N 片段拼进 prompt,交给 LLM 生成最终答案。

## 为什么能减少幻觉

幻觉的根因是模型在不确定时倾向"编造"。RAG 通过注入真实上下文,让模型基于检索到的证据回答,而不是仅凭参数记忆。当知识库覆盖问题域、切分与检索质量高时,幻觉率可大幅下降。

## 混合检索(向量 + 全文)

单一向量检索对长尾词、专有名词、代码符号等不敏感;全文检索(FTS/BM25)则擅长精确关键词命中。把两者结果做加权融合(俗称 hybrid search),既保留语义泛化,又不丢精确匹配,是工业级 RAG 的主流做法。重排阶段再统一打分,进一步压缩噪声。
