# 阶段总结报告 · AI 生成 prompt

## 用途
转型某阶段（或整体）结束的**总结报告**——成果、数据、经验教训、下阶段规划。
适合：阶段结项、晋升述职、季度/年度总结。

## 你要准备的数据
- **阶段成果**：完成了什么（阶段、动作、覆盖范围）
- **数据**：产出 / AI 占比 / Bug / 人均（同比，带前后数字）
- **经验教训**：做对的、踩的坑、沉淀的方法论
- **下阶段规划**：接下来做什么

## Prompt（复制下面的内容，填入你的数据，发给 AI）

```
你是一位技术战略顾问。根据我提供的数据，生成一份《AI 转型阶段总结报告》（可用于结项汇报或晋升述职）。

结构：
1. 执行摘要（一段话概括 + 3-5 个核心数据）
2. 目标达成情况（目标 vs 实际，表格）
3. 关键成果（分维度：产出 / 质量 / AI 渗透 / 能力建设，每维度带数据）
4. 数据明细（同比对比表）
5. 经验与教训（做对的 3 条、踩的坑 3 条、可复用的方法论）
6. 下阶段规划

风格：用数据说话、有结论、有沉淀；避免空话和夸大；数据要标注口径。

数据（替换方括号占位）：
- 阶段成果：[填完成了什么]
- 数据：[填同比数据，如人均产出+104%、AI占比76.6%、Bug-16pp]
- 经验教训：[填做对的、踩的坑]
- 下阶段：[填规划]

请直接输出完整的 Markdown 报告，不要解释。
```

## 输出示例（核心数据卡片，节选）
```markdown
## 一、执行摘要
本阶段完成全员 AI 赋能与闭环试点，转型从「口号」变为可量化能力。

> 📊 人均代码产出 **+104%** | AI 代码占比 **76.6%** | Bug 占比 **-16.4pp** | 测试用例生成 **5-10 倍**

## 二、目标达成
| 目标 | 计划 | 实际 | 达成 |
|---|---|---|---|
| 人均产出 | +50% | +104% | ✅ 超额 |
| AI 占比 | ≥70% | 76.6% | ✅ 达成 |
| Bug 占比 | -15pp | -16.4pp | ✅ 达成 |

## 五、经验与教训
**做对的**：① 思想转变先行（全员先过总纲）② 规范资产化（CLAUDE.md 提交 Git）③ 度量驱动（不靠感觉靠数据）
**踩的坑**：① 跳过探索直接编码（=重写 bug）② 全员开全权限（安全风险）③ CLAUDE.md 越写越臃肿
```

## 渲染
文档类（Markdown），直接用。适合转 PDF 作为正式结项文档/述职材料。
