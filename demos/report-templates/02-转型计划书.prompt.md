# 转型计划书 · AI 生成 prompt

## 用途
**立项审批**用的正式计划书——把 AI 转型作为项目立项，申请资源、明确目标与里程碑、识别风险。
适合：提交给 PMO/高管审批、作为项目正式启动文档。

## 你要准备的数据
- **现状与痛点**：当前研发效率/质量的具体问题（带数据）
- **转型目标**：总目标 + 分阶段可量化目标
- **范围**：覆盖哪些团队/项目/角色
- **里程碑**：各阶段时间节点 + 交付物
- **资源**：人力/工具预算/培训/基础设施
- **风险**：预判的风险 + 应对

## Prompt（复制下面的内容，填入你的数据，发给 AI）

```
你是一位资深的技术管理顾问。请根据我提供的数据，生成一份正式的《AI 战略转型计划书》（立项文档），供管理层审批。

结构要求：
1. 项目背景与现状（痛点 + 数据支撑）
2. 转型目标（总目标 + 3-5 个可量化分目标）
3. 转型范围（覆盖团队/项目/角色）
4. 实施路线（分阶段，每阶段：时间、目标、关键动作、交付物、负责人）
5. 里程碑表（时间节点 + 交付物）
6. 资源需求（人力/工具预算/培训/基础设施，列清单）
7. 风险与应对（风险矩阵：风险 + 概率 + 影响 + 应对）
8. 预期收益（量化）
9. 成功标准（如何判定转型成功）

风格：正式、简洁、可执行；多用表格和清单；避免空话。

数据（请用我提供的真实数据替换方括号占位）：
- 现状痛点：[填你的现状问题，带数据]
- 目标：[填你的转型目标]
- 范围：[填覆盖范围]
- 里程碑：[填你的时间节点]
- 资源：[填你的资源需求]
- 风险：[填你预判的风险]

请直接输出完整的 Markdown 计划书，不要解释。
```

## 输出示例（节选）
```markdown
## 一、项目背景与现状
- 需求积压：平均需求交付周期 X 天，业务方满意度低
- 重复劳动：CRUD/模板代码占开发量约 X%，大量人力消耗在低价值工作
- 竞对压力：行业普遍引入 AI 编码工具，提效显著

## 二、转型目标
| 目标 | 指标 | 目标值 |
|---|---|---|
| 提升产出 | 人均代码产出 | +50% |
| AI 渗透 | AI 代码占比 | ≥70% |
| 提速交付 | 交付周期 | -30% |
| 质量提升 | Bug 占比 | -15pp |

## 四、实施路线（6 阶段）
| 阶段 | 时间 | 目标 | 关键动作 |
|---|---|---|---|
| 战略启动 | 第1-3周 | 目标量化、组织就绪 | 定目标、组团队、排风险 |
| 全员赋能 | 第4-7周 | 全员会用 AI | 工具培训 100%、规范资产化 |
| ... | ... | ... | ... |
```

## 渲染
文档类（Markdown），直接用。可用 Typora/VS Code/飞书文档打开，或转 PDF。
