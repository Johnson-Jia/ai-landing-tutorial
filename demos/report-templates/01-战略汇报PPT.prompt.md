# 战略汇报 PPT · AI 生成 prompt

## 用途
向管理层/决策层汇报 AI 转型、争取资源/批准。**同一份 PPT，两个阶段用不同数据立论——先确认你处于哪个阶段**：

- **启动版（阶段 1 · 立项 / 要资源）**：还没试点，**没有验证成果数据** → 用「现状痛点 + 目标承诺 + 行业参考 + 试点计划」立论
- **成果版（阶段 5 · 推广 / 要继续投入）**：已试点，有实测数据 → 用「实测成果 + 推广计划」立论

> ⚠️ **别用错阶段的数据**：启动版**绝对不要**写「人均产出 +104%」「AI 占比 76.6%」这种验证成果——你还没跑出来，写了自己不信、一追问就穿帮。+104% 这类数据是阶段 5 用度量工具跑出来的**结果**，归成果版。启动版靠「现状有多痛 + 目标要达到 + 先试点验证再说」来支撑立项，更诚实也更有力。

## 你要准备的数据

**启动版（阶段 1）准备这些**——都是企业现在就有的，不需要等转型跑完：
- **现状痛点**（自有真实数据）：需求交付周期、重复劳动占比、Bug 率、积压需求数等
- **转型目标**（承诺，不是结果）：人均产出 +50%、AI 占比 ≥70%、交付周期 -30% 等
- **行业参考**（公开研究，**必须标注来源**）：证明目标可达，如行业 AI 提效报告（注明出处）
- **试点计划**：先在 1-2 个组试点 3 个月，用度量工具验证，再推广
- 资源请求 + 6 阶段路线图 + 你的角色

**成果版（阶段 5）准备这些**：
- **验证数据**（实测）：AI 占比、人均产出同比、Bug 下降等（来自 demos/ai-metrics）
- 推广计划 + 资源请求

## Prompt（复制下面的内容，填入你的数据，发给 AI）

```
你是一位资深的技术战略顾问，擅长把 AI 转型讲清楚、让管理层快速理解。请根据我提供的数据，生成一份《AI 战略转型汇报》的正文内容，用结构化 Markdown 格式（这份 Markdown 后续会交给 ppt-master 渲染成原生可编辑 PPT，所以你只管把内容和结构写清楚，排版交给渲染层）。

请先看我标注的【汇报场景】，按对应版本生成。

格式要求：
1. 不要写 frontmatter，不要用 HTML 注释（不要 <!-- --> 标记），不要用 --- 分页——这些 ppt-master 用不上
2. 用一级标题（#）写主标题，二级标题（##）分章节
3. 章节结构按场景走：

【启动版 · 阶段 1 立项要资源】
- 开头：主标题（#）+ 一句副标题 + 汇报人
- 为什么必须转：现状痛点（用企业自己的真实数据：交付周期 / 重复劳动 / Bug / 积压）
- 转型目标：要达成什么（表格，列名标「目标值」，不是已达成）
- 凭什么能做到：工具已成熟 + 行业公开提效研究（必须标注来源）
- 风险可控的验证路径：先 1-2 组试点 3 个月 → 用度量工具验证 → 再推广
- 6 阶段路线图（每阶段一行）
- 资源请求 + 请求批准

【成果版 · 阶段 5 推广汇报】
- 开头：主标题（#）+ 一句副标题 + 汇报人
- 转型目标 vs 实测达成（表格）
- 实测成果：验证数据用引用块(>)高亮关键数字，标注数据口径，并注明「相关性≠因果」
- 推广计划
- 资源请求 + 请求批准

4. 风格：数据用表格或引用块，避免大段文字；行业参考必须标注来源；启动版绝不编造未经验证的成果数据。

数据（请用我提供的真实数据替换方括号占位）：
- 汇报场景：[启动版（阶段1）/ 成果版（阶段5）]
- 现状痛点（启动版用）：[填企业自己的真实数据]
- 验证数据（成果版用，来自 ai-metrics）：[填]
- 转型目标：[填，如人均产出+50%、AI占比≥70%、周期-30%]
- 行业参考（启动版用，附来源）：[填公开研究 + 出处]
- 试点计划（启动版）/ 推广计划（成果版）：[填]
- 资源请求：[填]
- 我的角色：[填]

请直接输出 Markdown 全文，不要解释。
```

## 输出示例（启动版 · 阶段 1，全程无验证数据）
> 启动版该有的样子——**没有 +104% 这种验证成果**，靠现状、目标、行业参考、试点计划立论。

```markdown
# AI 战略转型汇报

面向管理层的转型立项与资源申请。

## 一、为什么必须转：现状有多痛
- 需求积压：平均交付周期 28 天，业务方满意度持续走低
- 重复劳动：CRUD / 模板代码约占开发量 40%，人力消耗在低价值工作
- 质量承压：Bug 占比居高，返工占用大量精力
- 行业倒逼：竞对普遍引入 AI 编码，不动即相对落后

## 二、转型目标（计划达成）
| 目标 | 指标 | 目标值 |
|---|---|---|
| 提升产出 | 人均代码产出 | +50% |
| AI 渗透 | AI 代码占比 | ≥70% |
| 提速交付 | 交付周期 | -30% |
| 质量提升 | Bug 占比 | -15pp |

## 三、凭什么能做到
- 工具已成熟：Claude Code / Codex 已具备端到端编码能力
- 行业参考：[填公开提效研究 + 出处，如某权威报告显示 AI 编码普遍提升 X%]

## 四、风险可控的验证路径
先在 1-2 个组试点 3 个月，用 AI 代码占比度量工具（三层识别防伪造）验证真实效果，**拿数据再推广**——不靠感觉靠证据。

## 五、6 阶段路线图
1. 总纲：思想转变 + 原理认知
2. 战略启动：目标量化 + 组织
3. 全员赋能：工具 + 培训 100%
4. 基础设施：RAG + 代码图谱（按需）
5. 闭环试点：OpenSpec + AI 测试
6. 推广度量：多 Agent + 效果度量

## 六、资源请求
工具预算 / 培训投入 / 基础设施人力 / 考核绑定。

## 七、请求批准
批准立项 + 试点资源。3 个月后用试点实测数据回来汇报，再决定是否全员推广。
```

## 渲染成 PPT
内容生成后，在 **Claude Code** 里用 [**ppt-master**](https://github.com/hugohe3/ppt-master) skill 生成**原生可编辑 PPTX**（不是图片式 PPT，PowerPoint 里每个元素都能改）。

**工作流**：
1. 把 AI 输出的汇报内容存为 `.md`
2. 安装 ppt-master（一次性）：
   ```bash
   git clone https://github.com/hugohe3/ppt-master.git
   cd ppt-master && pip install -r requirements.txt
   ```
3. 在 Claude Code 里 `@hugohe3/ppt-master`，说「**用这份战略汇报内容生成 PPT**」
4. AI 按 ppt-master 的 `SKILL.md` 工作流，生成原生可编辑 `.pptx`

> **关键**：PPT 类汇报用 ppt-master 生成原生可编辑 PPTX（DrawingML 形状/文本框），AI 生成内容 → ppt-master 生成精美 PPT，不用手画。ppt-master 的 [`examples/`](https://github.com/hugohe3/ppt-master) 目录有 20+ 精美可编辑 deck 可参考。
