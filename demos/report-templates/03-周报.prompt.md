# 周报 · AI 生成 prompt

## 用途
转型推进过程中的**关键节点跟踪**——每周/每月向上汇报进展、数据、问题、计划。
适合：转型期持续汇报，让管理层看到推进节奏与数据变化。

## 你要准备的数据
- **本周进展**：完成了哪些阶段/动作（如全员培训、试点启动）
- **关键数据**：AI 占比、培训覆盖率、试点用例数等（有就填）
- **问题/风险**：遇到的阻力、卡点
- **下周计划**：接下来做什么
- **需要支持**：需要管理层协调的事（如有）

## Prompt（复制下面的内容，填入你的数据，发给 AI）

```
你是 AI 转型项目的汇报人。根据我提供的数据，生成一份简洁的《AI 转型周报》，供管理层快速了解推进情况。

结构（一页内能看完）：
1. 本周进展（3-5 个要点）
2. 关键数据（表格：指标 | 本周值 | 趋势）
3. 问题与风险（问题 + 影响 + 需要的支持）
4. 下周计划（3-5 个要点）

风格：简洁、数据导向、一目了然。避免大段文字。

数据（替换方括号占位）：
- 本周进展：[填本周完成的事]
- 关键数据：[填 AI 占比/培训覆盖率/用例数等，没有就删掉这节]
- 问题：[填遇到的问题]
- 下周计划：[填下周要做的事]

请直接输出 Markdown 周报，不要解释。
```

## 输出示例（节选）
```markdown
## 本周进展
- ✅ 全员 Claude Code 培训完成，覆盖率 100%（82/82）
- ✅ 制定《AI 代码占比度量方案》，部署度量工具
- ✅ 试点组（A 组）启动 OpenSpec + Superpowers 工作流

## 关键数据
| 指标 | 本周 | 趋势 |
|---|---|---|
| 培训覆盖率 | 100% | ↑ |
| AI 代码占比（试点组） | 68% | ↑ |
| 试点用例 | 12 个 | ↑ |
```

## 渲染
文档类（Markdown），直接用。可贴飞书/钉钉/邮件，或转 PDF。
